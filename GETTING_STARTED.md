# 🎯 Getting Started

Быстрое руководство для начинающих пользователей.

## ⚡ 5-минутный старт

### 1. Требования

- Docker & Docker Compose установлены
- Токен бота Mattermost
- Доступ к Mail.ru календарю

### 2. Быстрая установка

```bash
# 1. Клонировать
git clone https://github.com/USERNAME/calendar_bot.git
cd calendar_bot

# 2. Создать .env
cp .env.example .env

# 3. Заполнить .env
nano .env
# Вставьте: MATTERMOST_BASE_URL, MATTERMOST_BOT_TOKEN, MM_ACTIONS_URL, ENCRYPTION_KEY

# 4. Запустить
docker-compose up -d

# 5. Проверить
docker-compose logs -f
```

Готово! 🎉

## 📱 Как использовать

### Первый запуск

1. **Откройте Mattermost**
2. **Создайте новое Direct Message**
3. **Напишите**: `@calendar_bot`
4. **Бот ответит** с инструкциями по авторизации

### Авторизация

1. Бот отправит ваш email
2. **Перейдите** на https://account.mail.ru/user/2-step-auth/passwords/
3. **Создайте** пароль приложения
4. **Отправьте** пароль боту
5. **Готово!** Теперь доступно главное меню

### Главное меню

После авторизации видите 4 кнопки:

```
📅 Все встречи на сегодня
   ↓
   Показывает таблицу встреч на сегодня
   Можете выбрать встречу для просмотра деталей

⏱️ Текущие встречи
   ↓
   Встречи, которые идят прямо сейчас
   + встречи до конца дня

➕ Создать встречу
   ↓
   Пошаговый мастер (7 вопросов)
   → Название
   → Дата (DD.MM.YYYY)
   → Время (HH:MM)
   → Продолжительность (минуты)
   → Участники (@username или email)
   → Описание (опционально)
   → Место/ссылка (опционально)

🚪 Разлогиниться
   ↓
   Удаляет все ваши данные
   При следующем обращении - авторизация заново
```

## ⚙️ Конфигурация

### Где найти нужные значения?

**MATTERMOST_BASE_URL**
- Откройте Mattermost
- URL в адресной строке (например: `https://mattermost.company.com`)

**MATTERMOST_BOT_TOKEN**
- Mattermost Admin Console
- System Console → Integrations → Bot Accounts
- Найдите `calendar_bot` → скопируйте Token

**MM_ACTIONS_URL**
- Адрес, на котором доступен бот
- Пример: `https://calendar-bot.company.com`

**ENCRYPTION_KEY**
- Запустите: `python generate_key.py`
- Скопируйте результат

### Опциональные переменные

```env
# Сервер календаря (по умолчанию Mail.ru)
CALDAV_BASE_URL=https://calendar.mail.ru

# Временная зона (по умолчанию Москва)
TZ=Europe/Moscow

# Как часто проверять изменения (секунды)
CHECK_INTERVAL=60

# За сколько минут напоминать о встречах
REMINDER_MINUTES=15
```

## 🆘 Частые вопросы

### Q: Бот не отвечает на сообщения
**A:** 
- Проверьте, что вы напишете `@calendar_bot` точно
- Убедитесь, что это личное сообщение (Direct Message)
- Проверьте логи: `docker-compose logs`

### Q: Ошибка "Invalid credentials"
**A:**
- Проверьте, что используете пароль **приложения**, а не пароль от аккаунта
- Пароль должен быть создан на https://account.mail.ru/user/2-step-auth/passwords/

### Q: Как разлогиниться?
**A:**
- Нажмите кнопку `🚪 Разлогиниться`
- При следующем обращении потребуется авторизация заново

### Q: Где хранятся мои пароли?
**A:**
- Пароли шифруются и хранятся в локальной БД
- Используется алгоритм Fernet (надежное шифрование)
- Пароли никогда не отправляются в логи

### Q: Можно ли удалить свои данные?
**A:**
- Да, нажмите `🚪 Разлогиниться`
- Все ваши данные и пароли будут удалены

### Q: Как обновить бота?
**A:**
```bash
git pull origin main
docker-compose up -d --build
```
